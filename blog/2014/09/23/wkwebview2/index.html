WKWebViewとUIWebViewのベンチマーク取ってみた(Google Octane2.0 利用)

Sep 23rd, 2014

WKWebViewとUIWebViewのベンチマーク比較

本日はGoogle Octane2.0を利用してWKwebViewとUIWebViewのベンチマークを取った結果を紹介したいと思います。
まずは、Google Octane2.0とは何かから説明しましょう。
Octane2.0とはGoogleからオープンソース提供している。ブラウザのパフォーマンスを測定するJavaScriptベンチマークスイートのことです。昨年、バージョン2.0が後悔され、これまでのテスト項目13から17への増えました。
では、そのテスト項目とは何があるのか説明しましょう。

  • Richards: OSカーネル・シミュレーション・ベンチマーク。Martin RichardsによってBCPLで書かれたもの。
  • Deltable: 単変数の制約解法。John MaloneyとMario WolczkoによってSmalltalk
  • Raytrace: Adam Burmisterのコードを元に作成されたレイトレースベンチ
  • Regexp: 正規表現ベンチ。最も人気のある50のWebページから正規表現の処理を抽出して作成。
  • NavierStokes: 二次ナビエ・ストークス方程式解法(倍精度配列を大量に処理)。Oliver Huntのコードを元にしている。
  • Crypto: Tom Wuのコードを元にして作成した暗号化と復号化ベンチ。
  • Splay: スプレー木や自動メモリ管理サブシステムを実行するデータ操作ベンチ。
  • SplayLatency: VMのGCサブシステムに圧力を与えるスプレー試験。
  • EarleyBoyer: 典型的なSchemeベンチをFlorian LoitschのScheme2Jsコンパイラを用いてJavaScriptに変換したもの。
  • pdf.js: JavaScriptで実装されたMozillaのPDFリーダ。デコード、インタープリター処理の時間を計測。
  • Mandreel: Mandreelを利用してC++からJavaScriptに移植した3D Bullet 物理エンジン。
  • MandreelLatency: スプレー試験に似ている、Mandreelベンチ試験。VMコンパイラーにMandreelが圧力を与えることで、発生する待ち時間の目安を計測する。
  • GB Emulator: 携帯用コンソールのアーキテクチャをエミュレートして、厳しい3DシミュレーションをJavaScriptで実行。
  • Code loading: 大規模なJavaScriptプログラムをロードしたあと、JavaScriptエンジンがどれだけ速くコードの実行を開始できるかを計測。
  • Box2DWeb: Erin Cattoによって書かれた人気の2D物理エンジンであるBox2DWebのJavaScript移植版がベース。
  • zlib: zlib asm.js/Emscripten試験。
  • Typescript: MicrosoftのTypescriptコンパイラは複雑なアプリケーションです。この試験はTypescriptがコンパイルする時間を計測します。

このOctane2.0を用いて、試験した結果を報告します。
今回は、下記条件で試験しました。

利用端末: iPhone5
利用OS: iOS7.1.2, iOS8.0
利用アプリ: Safari, 自作アプリ(内部にUIWebView利用), 自作アプリ(内部にWKWebView利用)

組み合わせは下記です。

  • Safari / iOS7.1.2
  • UIWebView / iOS7.1.2
  • Safari / iOS8.0
  • UIWebView / iOS8.0
  • WKWebView / iOS8.0

結果は下記になりました。

また、トータルスコアは下記になりました。
* Safari / iOS7.1.2: 2,642
* UIWebView / iOS7.1.2: 453
* Safari / iOS8.0: 2983
* UIWebView / iOS8.0: 447
* WKWebView / iOS8.0: 3097

全て1回だけ計測したため、ある程度の誤差があるかもしれません。
しかし、UIWebViewとWKWebViewが圧倒的に異なるということはわかりました。また、SafariとWKWebViewの結果がほぼ同じであることから、これまでSafariでしか利用できなかった各種機能が解放されていることがわかりました。
因みに、UIWebViewで計測するとき、Box2DWebまでは他と大きな差がなく進んだのですが、zlibとTypescriptの試験が非常に時間がかかりました。(もうこれ以上は計測できないのでは？と思ったほどです。)
